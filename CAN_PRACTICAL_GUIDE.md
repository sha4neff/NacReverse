# Побайтовое описание CAN-фреймов PSA NAC (AEE2010)

**Прошивка:** 44.07.33.32_NAC-r0, NAC_EUR_WAVE4
**Источник данных:** SensorFramework g_CanAppMsg_Table + NavLib initializeSignalIdHash() + анализ Logic-классов
**Всего фреймов:** 99 реальных CAN ID (RX), 3 TX CAN ID (мультиплекс), 927 сигналов
**Покрытие имён:** ~878/927 (95%) из NavLib + Ghidra-анализа Logic-классов + MASTER
**Оставшиеся ~49 «—»:** ~29 в BSI эхо (TX конфиг-флаги, не в SOC), ~7 в 0x220, ~13 прочие
**Семантика:** enum-значения, формулы масштабирования, описания — все 99 фреймов

**Обозначения:**
- **Байт.Бит** — позиция начала сигнала. **Нумерация бит MSB-first:** Бит 0 = старший бит байта (MSB, 0x80), Бит 7 = младший (LSB, 0x01). Перевод в стандартную нотацию: D[Байт] бит (7 − Бит)
- **Ш** — ширина в битах
- **BSI** — фрейм от BSI (эхо настроек TX), **EXT** — внешний фрейм (RX от шины)
- Имена без префикса `SIG_ID_` — из NavLib хеш-таблицы или Ghidra-анализа
- `—` — имя не найдено в прошивке
- **DONNEES_*** — PSA-имя фрейма из VuC таблиц

# Зависимости от конфигурации автомобиля

NAC использует **4-уровневую систему гейтирования** — фреймы и сигналы активны только при наличии опции:

## Уровень 1: ERG ключи (BSI EEPROM, телекодирование)

| ERG Key (ID) | Значения | Влияние |
|--------------|----------|---------|
| **OPTION_EXPORT_MEDIA** (0x100848E) | 0=Выкл, 1=Вкл | Медиа-экспорт на CAN 0x1B6 |
| **TYPE_EXP_MEDIA_CAN** (0x1008495) | 0=GEN1, 1=GEN2, 2=GEN3 | Размер текста: GEN1/2=60B, GEN3=90B. MirrorLink только GEN3 |
| **ERG_OVIP_OPTION_IDVR** (DiagServer 0x6F) | 0=Нет, 1=Есть | IDVR (видеорегистратор): все фреймы 0x1DC/0x25C/0x3C5/0x345/0x385 |
| **TPMS presence** (DiagServer 0x20) | 0=Нет, 1=Есть | TPMS: фрейм 0x1E1 (давление шин) |
| **DRIVER_REST thresholds** (0x100AC82/83) | 0-255 (÷100) | Порог усталости водителя (D-Bus, не CAN) |

## Уровень 3: PPS Features (телекодинг PolySensPack)

PPS имеет QMap<int,bool> с 10+ ключами. Signal_id 522 (PRES_AMBIANCE) содержит массив телекодинга.

| Feature | Назначение | Зависимые сигналы |
|---------|------------|-------------------|
| **Feature 8** | PPS selector mode | ON → sig 897 (CAN 0x3A3), OFF → sig 837 (CAN 0x268) |
| **Feature 10** | Dynamic mode | ON → sig 832 (CAN 0x268) регистрируется |
| **Feature 4** | Family Items | Массив до 39 записей из signal_id 522 |

## Уровень 4: Runtime-состояния

| Сигнал (signal_id) | Где | Влияние |
|---------------------|-----|---------|
| **MODE_HY** (481) | CAN RX | =1: гибрид активен → PowerFlow (0x31F), SOC_BATT (0x3E0), STT (0x3E1) |
| **ETAT_MONO** (689) | CAN 0x2E0 | =1: моно-зона → AVD (правая) температура синхронизирована с AVG |

## Конфигурационные профили (примеры)

**Базовый ICE (без опций):** ~60% фреймов активны. Нет: IDVR, PPS, TPMS, XVV, SAM, гибрид.
**Полный гибрид (все опции):** 100% фреймов активны. PowerFlow, SOC_BATT, STT, TPMS, XVV, PPS, IDVR.
**Городской авто (средний):** ~80%. XVV + TPMS + SAM, без IDVR/PPS/гибрид.

# ЧАСТЬ 1: TX — Фреймы передачи (NAC → CAN шина)

**348 TX сигналов** (signal_id 0–347) записываются приложениями SOC через D-Bus.
VuC упаковывает их в **32 внутренних фрейма**, мультиплексируемых на **3 физических CAN ID**:

| Физический CAN ID | Имя PSA | Период | Назначение |
|-------------------|---------|--------|------------|
| **0x2A2** | DEMANDES_BTEL | 200ms | Основной TX: настройки, навигация, XVV, аудио |
| **0x1B6** | DEMANDES_BTEL2 | 200ms | Вторичный TX: климат, PPS, дисплей |
| **0x305** | IDVR_DEMANDES_BTEL | 200ms | TX видеорегистратора (IDVR) |

> **Важно:** Каждый из 3 CAN ID несёт разные данные в разное время (TDM мультиплексирование).
> Побайтовый формат зависит от того, какой из 32 внутренних фреймов VuC отправляет в данный момент.
> BSI получает TX данные и отправляет **эхо** обратно на отдельных CAN ID (0x01C, 0x069 и т.д.) —
> они описаны в Части 2 (RX) с пометкой «BSI».
